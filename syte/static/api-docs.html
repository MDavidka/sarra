Authentication

All endpoints below require an API key except POST /api/tokens and GET /api/health.

1. Create an API token

In the Syte GUI: key in the navbar → create token. Or via API (no auth):

curl -X POST https://YOUR_HOST/api/tokens \
  -H "Content-Type: application/json" \
  -d '{"name": "my-ai-agent"}'

# Response (save token immediately — shown once):
# {"ok":true,"token":"syte_abc123...","prefix":"syte_abc..."}
2. Send the token on every request

Option A — header (recommended):

X-API-Key: syte_your_token_here

Option B — Bearer:

Authorization: Bearer syte_your_token_here
curl https://YOUR_HOST/api/workspace_list \
  -H "X-API-Key: syte_your_token_here"

401 missing_api_key — no header sent
401 invalid_api_key — wrong or revoked token

Sycord Design Contract

Mandatory UI rules for every AI-generated site. Read GET /api/ai.json → system_prompt and design_contract before generating.

  • Next.js + shadcn/ui + Tailwind + Lucide icons
  • Font: Inter via --font-sans on body
  • shadcn theme preset + one accent; dark mode required
  • Hero gradient below hero section (primary at 0.15–0.3 opacity)
  • No icons in colored circles; no npm run build via execute_command

After generation: GET /api/validate_design?uuid= — automated linter

Live Preview API

Fast dev server with hot reload (HMR) — starts in seconds. Preview uses your connected HTTPS domain via Caddy (not raw IP:port).

Preview URL (connected domain)

Connected domainPreview URL
Project domain app.example.comhttps://preview.app.example.com
GUI domain only sycord.sitehttps://preview-{uuid}.sycord.site
No domain configuredFalls back to http://IP:4000+ (preview_direct_url)

DNS: add an A record for the preview hostname pointing to your server IP (same as production). Caddy issues TLS automatically.

Workflow

  1. POST /api/set_domain — connect project domain (recommended before preview)
  2. POST /api/start_preview {"uuid":"..."}
  3. Poll GET /api/preview_status?uuid= until preview_ready: true
  4. Open preview_url (HTTPS domain) — edits via write_file hot-reload
  5. POST /api/stop_preview when done
  6. POST /api/issue_deploy for production deploy

Start preview

curl -X POST https://YOUR_HOST/api/start_preview \
  -H "X-API-Key: syte_..." -H "Content-Type: application/json" \
  -d '{"uuid": "my-site-abc123"}'

Requirements

  • package.json must have a "dev" script (next dev or vite)
  • Run npm install via execute_command before first preview
  • Preview ≠ production — use issue_deploy to build for production

Deploy rules

Build ONLY via POST /api/issue_deploy {"uuid":"..."} — git pull + docker build + restart.

execute_command is for scaffolding, npm install, and npm run lint (testing). npm run build is blocked.

AI workflow: deploy from Git

  1. POST /api/create_project with git_url → uuid
  2. POST /api/issue_deploy {"uuid":"..."} → git pull + docker build + start
  3. Stream GET /api/projects/{uuid}/logs/stream?live=1 (SSE)
  4. GET /api/validate_design?uuid= → design linter
  5. POST /api/set_domain → optional HTTPS domain
curl -X POST https://YOUR_HOST/api/issue_deploy \
  -H "X-API-Key: syte_..." -H "Content-Type: application/json" \
  -d '{"uuid": "my-site-abc123"}'

AI workflow: build from scratch (no git)

  1. Read /api/ai.json → system_prompt + design_contract
  2. POST /api/create_project with name only → uuid
  3. POST /api/write_file — Next.js + shadcn/ui per design contract
  4. POST /api/execute_command — npm install, npm run lint (NOT build)
  5. GET /api/validate_design?uuid= — check design rules
  6. POST /api/issue_deploy {"uuid":"..."} — docker build + start
  7. GET /api/get_logs?uuid= — verify deploy
curl -X POST https://YOUR_HOST/api/execute_command \
  -H "X-API-Key: syte_..." -H "Content-Type: application/json" \
  -d '{
    "uuid": "my-site-abc123",
    "command": "npm run lint",
    "cwd": "app",
    "timeout": 300
  }'

Endpoints

GET /api/server_info

Server public IP, version, base URLs.

GET /api/workspace_list

All projects with uuid, url, port, status, running.

GET /api/workspace_get?uuid=

Full project detail: urls, env, git, stream_url.

GET /api/list_files?uuid=&path=

List files/dirs. path relative to workspace (default root).

POST /api/execute_command no build

Scaffolding and lint testing only. npm run build is blocked — use issue_deploy.

fieldtypedescription
uuidstringrequired
commandstringshell command (no build)
cwdstringdefault app
timeoutintseconds, default 300, max 1800
envobjectextra env vars for this run

Allowed: npm install, npm run lint, ls -la, mkdir -p src

Forbidden: npm run build, yarn build, next build

{"ok":true,"exit_code":0,"output":"...","command":"npm run lint"}
POST /api/execute_commands

Run multiple commands in sequence. Stops on first failure by default.

{"uuid":"...","commands":[
  {"command":"npm install","cwd":"app"},
  {"command":"npm run lint","cwd":"app","timeout":300}
]}
POST /api/start_preview

Start fast dev server (next dev / vite) with HMR. Returns HTTPS preview_url on connected domain.

{"uuid": "my-site-abc123"}

See Live Preview API for full response fields and DNS setup.

GET /api/preview_status?uuid=

Poll until preview_ready: true. Returns preview_url, preview_domain, preview_direct_url.

POST /api/stop_preview 
{"uuid": "my-site-abc123"}

Stops dev server and removes Caddy preview route.

GET /api/projects/{uuid}/preview/logs/stream?live=1

SSE stream of preview dev server output. Optional api_key query param.

GET /api/validate_design?uuid=

Run Sycord Design Contract linter — checks shadcn/ui, --font-sans, Lucide, dark mode, hero gradient, etc.

{"ok":true,"passed":7,"total":8,"checks":[...]}

Returns content (utf-8) or base64 for binary.

multipart: uuid, path, file

POST /api/delete_file

POST /api/create_project

Creates an empty project. Git and files are optional — add anytime, deploy anytime.

fieldrequireddescription
nameyesdisplay name (only required field)
uuidnocustom id (auto-generated if omitted)
git_urlnocloned on issue_deploy, not required at create
git_providernoshorthand: github.com/user/repo.git
branchnodefault main
start_commandnoshell fallback if no Dockerfile
domainnocustom HTTPS domain
env_varsnokey-value env
deploynodefault false — set true to deploy immediately

Response includes design_contract, uuid, execute_command.body, issue_deploy.body, validate_design.
