Turso (agent sessions) libSQL — durable activity storage

Every agent turn (request, plan, tool calls, reply) is written to a durable Turso session keyed by UUID instead of a live stream. Clients fetch the session by id from the API. Leave blank to keep using local-only activity snapshots.

Optional — from the Turso dashboard. libsql:// pastes are rewritten to HTTPS automatically (required for AWS-hosted DBs).
Required for remote Turso databases.
